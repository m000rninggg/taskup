# Справка пользователя по работе с системой TaskUp

## 1. Назначение системы

TaskUp представляет собой веб-приложение для организации командной работы над проектами. Система объединяет управление командами, проектами, задачами, проектной документацией и аналитикой выполнения работ в едином рабочем пространстве.

Основная цель системы - упростить планирование, распределение и контроль задач внутри команды. Пользователь может создавать команды, подключать участников, создавать проекты, вести Kanban-доску, фиксировать проектную документацию, оставлять комментарии к задачам и отслеживать прогресс выполнения проекта.

**Скриншот 1. Главная страница сайта TaskUp**

## 2. Роли и доступ пользователей

В системе предусмотрены авторизованные и неавторизованные пользователи.

Неавторизованный пользователь может просматривать главную страницу, перейти к регистрации или входу в систему.

Авторизованный пользователь получает доступ к личному кабинету, списку команд, списку проектов, рабочему пространству проекта, задачам, документации, аналитике и профилю.

Владелец команды имеет расширенные права: он может редактировать команду, добавлять и удалять участников, создавать проекты для команды, изменять настройки проекта и удалять связанные объекты.

## 3. Главная страница

Главная страница знакомит пользователя с возможностями системы. На ней представлены преимущества TaskUp и основные направления работы:

| Блок | Описание |
|---|---|
| Приветственный блок | Кратко описывает назначение системы и предлагает начать работу |
| Преимущества | Показывает сильные стороны системы: работа в командах, аналитика, простота адаптации, оптимизация процессов |
| Команды | Описывает возможность создания команд и добавления участников |
| Kanban-доски | Описывает управление задачами проекта через доску |
| Документация | Описывает ведение проектной информации в отдельном разделе |

На главной странице доступны кнопки перехода к регистрации, личному кабинету, созданию команды, проектам и документации. Если пользователь не авторизован, система перенаправляет его на регистрацию.

**Скриншот 2. Блок возможностей на главной странице**

## 4. Регистрация и авторизация

Для начала работы пользователь должен создать учетную запись или войти в существующую.

### 4.1. Регистрация

Страница регистрации используется для создания новой учетной записи. Пользователь вводит данные, необходимые для создания профиля, после чего получает доступ к закрытым разделам системы.

**Скриншот 3. Форма регистрации пользователя**

### 4.2. Вход в систему

Страница входа предназначена для авторизации зарегистрированного пользователя. После успешного входа пользователь перенаправляется в личный кабинет.

**Скриншот 4. Форма входа в систему**

### 4.3. Восстановление пароля

Если пользователь забыл пароль, он может перейти на страницу восстановления пароля. Система отправляет ссылку для сброса пароля на адрес электронной почты пользователя.

**Скриншот 5. Форма восстановления пароля**

## 5. Личный кабинет

Личный кабинет является стартовой страницей авторизованного пользователя. В нем отображается персональная сводка по текущей работе.

На странице кабинета представлены:

| Элемент | Назначение |
|---|---|
| Приветствие | Отображает имя пользователя и текущую дату |
| Ближайшие дедлайны | Показывает задачи пользователя с ближайшими сроками выполнения |
| Мои задачи | Показывает список активных задач, назначенных пользователю |
| Быстрые действия | Позволяют перейти к проектам, командам и профилю |

Через личный кабинет пользователь может быстро перейти к задачам конкретного проекта, открыть список проектов, список команд или страницу профиля.

**Скриншот 6. Личный кабинет пользователя**

## 6. Управление командами

Раздел «Команды» предназначен для создания команд и управления их участниками. В списке команд отображаются название команды, участники, владелец и количество проектов.

### 6.1. Создание команды

Для создания команды пользователь нажимает кнопку «Создать команду». После этого открывается модальное окно создания команды.

В модальном окне заполняются следующие поля:

| Поле | Описание |
|---|---|
| Название команды | Обязательное поле с названием новой команды |
| Имена пользователей | Список username участников, которых нужно добавить в команду; несколько имен вводятся через запятую |

После нажатия кнопки «Создать» команда добавляется в список.

**Скриншот 7. Модальное окно создания команды**

### 6.2. Редактирование команды

Владелец команды может открыть модальное окно редактирования команды. В нем можно изменить название команды и добавить нового участника по username.

В окне также отображается список участников команды. Для каждого участника показываются имя и username. Владелец команды отмечается отдельной меткой.

**Скриншот 8. Модальное окно редактирования команды**

### 6.3. Удаление участника команды

Владелец команды может удалить участника из команды через список участников в модальном окне редактирования. Для удаления используется кнопка с иконкой корзины рядом с участником.

Владелец команды не может быть удален как обычный участник.

### 6.4. Удаление команды

Владелец может удалить команду. Перед удалением открывается модальное окно подтверждения. В нем отображается предупреждение о том, что команда и связанные с ней проекты будут удалены.

**Скриншот 9. Модальное окно подтверждения удаления команды**

### 6.5. Выход из команды

Если пользователь является участником команды, но не ее владельцем, он может выйти из команды с помощью кнопки «Выйти из команды».

## 7. Управление проектами

Раздел «Проекты» содержит список проектов, доступных пользователю. Для каждого проекта отображаются название, описание, связанная команда и количество задач.

### 7.1. Создание проекта

Для создания проекта пользователь нажимает кнопку «Создать проект». После этого открывается модальное окно создания проекта.

Поля модального окна:

| Поле | Описание |
|---|---|
| Название проекта | Обязательное название проекта |
| Команда | Команда, к которой будет привязан проект |
| Описание проекта | Краткое описание целей и содержания проекта |

После создания проект появляется в общем списке проектов.

**Скриншот 10. Модальное окно создания проекта**

### 7.2. Просмотр проекта

Кнопка «Просмотреть» открывает рабочее пространство проекта. По умолчанию пользователь может перейти к задачам проекта.

### 7.3. Редактирование проекта

Владелец команды может перейти к настройкам проекта через кнопку редактирования. В настройках можно изменить название и описание проекта, а также добавить участников.

### 7.4. Удаление проекта

Владелец команды может удалить проект. Перед удалением открывается модальное окно подтверждения, в котором указывается, что проект и связанные с ним данные будут удалены.

**Скриншот 11. Модальное окно подтверждения удаления проекта**

## 8. Рабочее пространство проекта

Рабочее пространство проекта содержит боковую навигацию и основные разделы проекта:

| Раздел | Назначение |

| Аналитика | Просмотр прогресса выполнения проекта |

**Скриншот 12. Рабочее пространство проекта**

| Название проекта | Основное название выбранного проекта |

## 10. Управление задачами

Раздел «Задачи» предназначен для планирования и контроля выполнения работ. Пользователь может переключаться между двумя представлениями: таблицей и Kanban-доской.

### 10.1. Представление «Таблица»

В табличном виде задачи отображаются строками. Для каждой задачи показываются:

| Поле | Описание |
|---|---|
| Исполнитель | Пользователь, связанный с задачей |
| Задача | Название и описание задачи |
| Статус | Текущее состояние выполнения |
| Срок выполнения | Дата создания и дедлайн либо сообщение об отсутствии срока |
| Действия | Редактирование или удаление задачи |

**Скриншот 14. Табличное представление задач**

### 10.2. Представление «Kanban-доска»

Kanban-доска разделяет задачи по статусам:

| Статус | Назначение |
|---|---|
| Идея | Задачи, которые еще не начаты |
| В разработке | Задачи, находящиеся в работе |
| В тесте | Задачи, проходящие проверку |
| Готово | Завершенные задачи |

Пользователь может создавать задачи в нужной колонке и изменять их статус.

**Скриншот 15. Kanban-доска проекта**

### 10.3. Создание задачи

Для создания задачи пользователь нажимает кнопку добавления задачи. В зависимости от места открытия формы задача создается с выбранным статусом.

Поля модального окна:

| Поле | Описание |
|---|---|
| Название задачи | Обязательное название задачи |
| Описание задачи | Дополнительное описание задачи |
| Срок выполнения | Дедлайн задачи |

**Скриншот 16. Модальное окно создания задачи**

### 10.4. Редактирование задачи

Модальное окно редактирования задачи позволяет изменить:

| Поле | Описание |
|---|---|
| Название задачи | Заголовок задачи |
| Описание задачи | Подробное содержание задачи |
| Статус | Один из статусов Kanban-доски |
| Срок выполнения | Дата завершения задачи |

После сохранения изменения отображаются в таблице и на Kanban-доске.

**Скриншот 17. Модальное окно редактирования задачи**

### 10.5. Удаление задачи

Удаление задачи доступно через меню действий. Перед удалением система выводит подтверждение действия.

### 10.6. Комментарии к задаче

Для обсуждения задачи используется модальное окно комментариев. В нем отображаются:

| Элемент | Описание |
|---|---|
| Автор задачи | Имя пользователя, создавшего задачу |
| Название и статус задачи | Основная информация о задаче |
| Описание | Текстовое описание задачи |
| Срок выполнения | Дедлайн или сообщение об отсутствии срока |
| Список комментариев | Все комментарии пользователей к задаче |
| Форма комментария | Поле ввода нового комментария |

Пользователь вводит комментарий в текстовое поле и отправляет его с помощью кнопки отправки.

**Скриншот 18. Модальное окно комментариев к задаче**

## 11. Документация проекта

Раздел «Документация» предназначен для хранения текстовой информации по проекту. Документы сгруппированы по категориям:

| Категория | Назначение |
|---|---|
| Основное | Главные материалы проекта |

Каждый документ отображается в виде раскрываемой карточки. В карточке показываются название, дата обновления, содержание и автор последнего изменения.

**Скриншот 19. Раздел документации проекта**

### 11.1. Создание темы документации

Для создания новой темы пользователь нажимает кнопку «Добавить тему». После этого открывается модальное окно создания темы.

Поля модального окна:

| Поле | Описание |
|---|---|
| Название темы | Обязательное название документа |
| Категория | Основное или дополнительное |
| Описание | Текстовое содержание документа |

**Скриншот 20. Модальное окно создания темы документации**

### 11.2. Редактирование темы документации

Редактирование темы позволяет изменить название, категорию и содержание документа.

**Скриншот 21. Модальное окно редактирования темы документации**

### 11.3. Удаление темы документации

Удаление темы выполняется через действия документа. После удаления тема исчезает из списка документации проекта.

## 12. Аналитика проекта

Раздел «Аналитика» показывает состояние выполнения проекта. На странице отображаются:

| Показатель | Описание |
|---|---|
| Прогресс проекта | Процент выполненных задач |
| Выполнено задач | Количество завершенных задач относительно общего количества |

| Всего задач | Общее количество задач проекта |
| В работе | Количество задач со статусом «В разработке» |
| Завершено | Количество задач со статусом «Готово» |

Прогресс рассчитывается как отношение количества завершенных задач к общему количеству задач проекта.

**Скриншот 22. Страница аналитики проекта**

После изменения данных пользователь нажимает кнопку сохранения.

**Скриншот 23. Настройки основной информации проекта**

**Скриншот 24. Управление участниками проекта**

## 14. Профиль пользователя

Раздел профиля предназначен для управления учетной записью пользователя.

На странице доступны следующие функции:

| Функция | Описание |

| Изменение пароля | Смена текущего пароля на новый |

**Скриншот 25. Модальное окно удаления учетной записи**

## 15. Модальные окна системы

В системе используются модальные окна для выполнения действий без перехода на отдельную страницу.

| Модальное окно | Где открывается | Назначение |
|---|---|---|
| Создание команды | Раздел «Команды» | Создание новой команды и добавление участников |
| Редактирование команды | Карточка команды | Изменение названия команды, добавление и удаление участников |
| Удаление команды | Карточка команды | Подтверждение удаления команды |
| Создание проекта | Раздел «Проекты» | Создание проекта и привязка его к команде |
| Удаление проекта | Карточка проекта | Подтверждение удаления проекта |
| Создание задачи | Раздел «Задачи» | Добавление новой задачи в проект |
| Редактирование задачи | Карточка или строка задачи | Изменение названия, описания, статуса и срока |
| Комментарии задачи | Карточка задачи | Просмотр информации о задаче и добавление комментариев |
| Создание темы документации | Раздел «Документация» | Добавление нового документа проекта |

| Удаление учетной записи | Профиль пользователя | Подтверждение удаления профиля |

## 16. Рекомендуемый порядок работы пользователя

1. Зарегистрироваться или войти в систему.
2. Перейти в раздел «Команды» и создать команду.
3. Добавить участников команды по username.
4. Перейти в раздел «Проекты» и создать новый проект для выбранной команды.
5. Открыть проект и перейти в раздел «Задачи».
6. Создать задачи, указать описание и сроки выполнения.
7. Обновлять статусы задач по мере выполнения.
8. Использовать комментарии для обсуждения задач.
9. Заполнять документацию проекта в разделе «Документация».
10. Отслеживать прогресс проекта в разделе «Аналитика».
11. При необходимости изменять данные проекта и участников в настройках.

## 17. Заключение

Система TaskUp предоставляет пользователю единое пространство для управления командной работой. Основные функции сайта включают регистрацию и авторизацию, управление командами, создание проектов, ведение задач на Kanban-доске, работу с документацией, обсуждение задач через комментарии и просмотр аналитики проекта. Благодаря использованию модальных окон основные действия выполняются быстро, без необходимости покидать текущую страницу.
